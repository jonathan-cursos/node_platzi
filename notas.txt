Rutas y capa de red: Responsabilidades y límites

-Vamos a crear la carpeta 'components', y crear el componente 'message', aquí, haremos el network, ademas,
  de la capa de red crearemos 'routes' y modificaremos el server.

network: Encargado de recibir la petición, procesar la info. y envarla al controlador.
  -Aquí tendremos los metodos del router que teníamos en el server, ya no los tendremos ahí.
  -Debemos crear un router para usar esos metodos.
  -Exportaremos el router por default (module.exports).

routes: Este será un router, pero lo llamaremos routes para no confundirlo con el router http.
  -Taerá el network de cada componente.
  -Exportaremos por default una función, que tendrá 'server' por parametro. 
  -Usará el 'use()' de este, y como parametros la ruta donde este correrá, y el network del componente que 
    correrá ahí. Esto lo haremos por cada componente. Esto hace que cuando se acceda a esa ruta, se llame
    a ese componente.
  -Nota: Es por esto, que en el network, en los metodos, ya podemos usar simplemente '/'.

server:
  -El router usado ya no será el de express, sino el de 'routes'.
  -Invocaremos su función, que tendrá el server como parametro.
  -El use() que tenía el router, ya no se necesita.
